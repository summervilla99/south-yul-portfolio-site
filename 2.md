# PART 2. 화면 설계(UI / UX Specification)

---

# 1. 전체 사이트 구조 (Information Architecture)

사이트는 하나의 브랜드 경험을 제공하는 것을 목표로 한다.

메뉴는 최소화하며 사용자가 길을 잃지 않도록 한다.

```
Landing

↓

ACT

↓

PHOTOS

↓

CONTACT
```

별도의 로그인이나 서브페이지는 존재하지 않는다.

사용자는 언제든 상단 Navigation을 통해 원하는 섹션으로 이동할 수 있다.

---

# 2. Navigation

## 위치

Desktop

상단 중앙 고정

```
ACT      PHOTOS      CONTACT
```

Landing에서도 항상 보인다.

스크롤 이후에도 상단에 유지된다.

(position: fixed)

---

## 디자인

* 흰색 텍스트
* 폰트 굵기 400~500
* Letter spacing 약간 넓게
* Hover 시 opacity 60%
* Active 메뉴는 opacity 100%

메뉴는 절대 크거나 화려하지 않다.

---

## Mobile

상단 고정

햄버거 메뉴는 사용하지 않는다.

3개의 메뉴만 존재하므로 그대로 노출한다.

---

# 3. Landing Hero

## 목적

배우의 첫인상을 전달한다.

텍스트보다 영상이 먼저 기억되어야 한다.

---

## Hero Layout

전체 화면을 영상으로 채운다.

```
┌───────────────────────────────┐

          ACT PHOTOS CONTACT

               (video)

               (video)

               (video)

             NAM YUL

└───────────────────────────────┘
```

---

## Hero Video

조건

* autoplay
* muted
* playsInline
* loop

영상은 Cover 방식으로 화면을 채운다.

여백이 생기면 안 된다.

---

## 영상 개수

v1

대표 영상 1개

v2 확장

3~5개의 Hero 영상을 랜덤 재생

또는

페이지 진입 시 랜덤 선택

---

## Overlay

영상 위에

Black

Opacity

15~25%

정도의 오버레이를 추가한다.

영상이 너무 밝아 메뉴가 보이지 않는 상황을 방지한다.

---

## Landing Text

하단 중앙

```
NAM YUL

Actor
```

매우 작게.

과하게 존재감을 드러내지 않는다.

---

## 스크롤 UX

Landing에서 아래로 스크롤하면

ACT가 자연스럽게 이어진다.

페이지가 끊기는 느낌을 만들지 않는다.

---

# 4. ACT Section

## 목적

출연작 소개

사진보다 작품이 먼저 기억되어야 한다.

---

## Layout

한 작품이 거의 한 화면을 차지한다.

```
────────────────────

큰 작품 이미지

────────────────────

제목

연도

감독

배역

설명

────────────────────
```

---

## 구성 요소

대표 이미지

↓

작품명

↓

연도

↓

감독

↓

배역

↓

짧은 설명

---

## Hover

이미지는 확대하지 않는다.

Opacity만 아주 약하게 변화.

---

## 설명 길이

최대

3줄

너무 긴 설명 금지.

---

## 작품 데이터 구조

```
title

year

director

role

description

thumbnail

optional:

youtube

poster

```

---

## 향후 확장

Trailer 버튼

IMDb 링크

PlayDB 링크

영화제 수상

추후 쉽게 추가 가능하도록 설계한다.

---

# 5. PHOTOS Section

## 목적

배우 프로필 사진

스틸

무드 사진

촬영 현장

등을 보여준다.

---

## Layout

Desktop

4~6열 Grid

```
□ □ □ □

□ □ □ □

□ □ □ □
```

간격은 일정하게 유지한다.

---

## Mobile

2열

```
□ □

□ □

□ □
```

---

## 사진 비율

원본 비율을 최대한 유지한다.

강제 Crop은 최소화한다.

---

## Interaction

사진 Hover

Opacity

98%

Shadow 없음

Scale 없음

과한 효과 금지.

---

## Click

사진 클릭

↓

Fullscreen Lightbox

↓

좌우 이동 가능

↓

ESC 닫기

↓

모바일 Swipe 지원

---

## Lazy Loading

첫 화면에 보이는 사진만 우선 로드한다.

나머지는 스크롤 시 로드.

---

# 6. CONTACT

CONTACT는

새 페이지가 아니다.

Modal이다.

---

## Modal Layout

```
Contact

Name

Email

Message

[ Cancel ]

[ Send ]
```

---

## 입력값

이름

이메일

메시지

---

## 전송

Resend API

↓

Next API Route

↓

관리자 이메일

---

## Success

모달 닫지 않는다.

대신

```
Thank you.

메일이 성공적으로 전송되었습니다.
```

메시지를 보여준다.

---

## Error

```
메일 전송에 실패했습니다.

잠시 후 다시 시도해주세요.
```

---

## Spam 방지

이메일 검증

필수 입력

추후 reCAPTCHA 추가 가능하도록 설계

---

# 7. Footer

Footer는 매우 단순하다.

```
© 2026 NAM YUL

Instagram

YouTube

Email
```

SNS 아이콘보다 텍스트 링크를 우선한다.

---

# 8. Page Transition

페이지 이동은

0.3~0.5초

Fade

정도만 사용한다.

슬라이드

Zoom

Parallax

등은 사용하지 않는다.

---

# 9. Loading Strategy

Landing Video는

Poster Image

↓

Video

순서로 표시한다.

영상 로딩 중 흰 화면이 나타나면 안 된다.

---

# 10. 사용자 경험 원칙

사용자는

"사이트가 예쁘다."

보다

"배우 분위기가 좋다."

를 먼저 느껴야 한다.

모든 UI는 이를 위해 존재한다.
